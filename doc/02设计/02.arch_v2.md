# BugVault v2 架构说明书

> 基于「多智能体系统 + 高级 RAG 架构 + MCP 协议」深度重构的生产级代码架构。
> 版本: v2.0 | 最后更新: 2026-05-29

---

## 一、 架构总览

BugVault v2 从双工具架构升级为**三大核心工具**架构，实现了 Agent 自进化能力的完整闭环：

| 阶段 | 工具 | 职责 |
|------|------|------|
| 🛠️ 排障中 | `retrieve_bug_experience` | 从历史库检索相似 Bug 经验，附带 RAG 可信度评分 |
| 💾 排障后 | `save_bug_experience` | 零阻塞异步沉淀：同步归档 Markdown → 后台异步 Embedding + 向量入库 |
| 🧠 复盘反思 | `reflect_and_prevent_error` | **新增** — 将预防规则写入 CLAUDE.md，形成防重犯记忆 |

### 核心设计目标

- **零阻塞保存**：保存工具仅做 Markdown 文件写入即返回，Agent 无需等待 Embedding 推理
- **检索质量评估**：可选的外部裁判 LLM 对召回结果进行 RAGAS 三元组评分
- **Agent 自进化**：复盘工具通过修改 CLAUDE.md 实现静态上下文的动态更新
- **严格层级依赖**：`main → mcp_tools → services → database`，消除反向依赖

## 三、 层级依赖与调用流向

### 3.1 严格单向依赖

```
main.py
  │
  ├─→ database/lancedb_client.py       [无 services 依赖]
  ├─→ services/embedding_svc.py        [无 database 依赖]
  ├─→ services/rag_evaluator_svc.py    [无 database 依赖]
  └─→ mcp_tools/tools.py
         │
         ├─→ services/archive_svc.py
         ├─→ services/ingestion_svc.py
         ├─→ services/retrieval_svc.py
         ├─→ services/reflection_svc.py
         ├─→ services/rag_evaluator_svc.py
         ├─→ services/embedding_svc.py
         └─→ database/lancedb_client.py
```

**关键改进**：`database/` 层不再导入任何 `services/` 模块，彻底消除了 v1 中的反向依赖。

### 3.2 三大工具调用流向

#### 保存工具 — 零阻塞异步飞轮

```
Agent → save_bug_experience
  │
  ├─ SYNC 路径（executor 线程）──────────── 毫秒级
  │   Pydantic 校验 → BugRecord
  │   validate_and_prepare()
  │   archive_svc.write_markdown_archive()  → 写入 .md 文件
  │   → 立即返回 "saved successfully"
  │
  └─ ASYNC 钩子（fire-and-forget）──────── 后台静默
      embedding_svc.generate_embedding()   → ONNX 推理
      db.upsert_record()                   → LanceDB 无锁写入
      异常仅记录日志，不阻塞 Agent
```

#### 查询工具 — 纯后端闭环质量把关

```
Agent → retrieve_bug_experience
  │
  ├─ embedding_svc.generate_embedding(query)
  ├─ db.search(embedding)                  → Top-K ANN 检索
  ├─ rerank(results)                        → 混合重排（语义×时间衰减）
  ├─ [可选] rag_evaluator.evaluate()       → 外部 LLM 评分
  │  返回 RAGEvalResult { score, evaluation }
  └─ 格式化文本 + 元数据 → Agent
```

#### 复盘工具 — 静态上下文动态更新

```
Agent → reflect_and_prevent_error
  ├─ 读取 CLAUDE.md
  ├─ 计数现有规则
  ├─ 追加新条目到 ## Bug Prevention Rules
  ├─ 写回 CLAUDE.md
  └─ 返回元数据 { rule_number, total_rules, category, created_at }
```

## 五、 新增模块详述

### 5.1 `services/archive_svc.py` — Markdown 归档服务

从 `ingestion_svc.py` 和 `lancedb_client.py` 提取出的纯 I/O 服务，职责单一。

- `record_to_markdown(record) → str` — 生成 YAML frontmatter + Markdown 正文
- `write_markdown_archive(record) → Path` — 写入归档目录，返回文件路径

### 5.2 `services/embedding_svc.py` — Embedding 模型封装

从 `lancedb_client.py` 提取，封装 `fastembed.TextEmbedding` 的初始化和推理。

- `EmbeddingService.__init__()` — 加载模型 + ONNX 热身
- `generate_embedding(text) → list[float]` — 生成稠密向量

### 5.3 `services/reflection_svc.py` — 复盘规则管理

- `ReflectionService(claude_md_path=None)` — 可指定路径，默认 `~/.bugvault/CLAUDE.md`
- `add_preventive_rule(reflection, category, rule) → dict` — 追加规则并返回元数据
- CLAUDE.md 格式：
  ```markdown
  ## Bug Prevention Rules

  - **Rule #1** [code_logic_error] (2026-05-29 07:40 UTC)
    - **Analysis**: 错误分析
    - **Prevention**: 预防规则
  ```

### 5.4 `services/rag_evaluator_svc.py` — RAG 质量评估

- `RAGEvaluator()` — 从 config 读取评估配置
- `evaluate_sync(query, results) → RAGEvalResult` — 同步评估（用于 executor 线程）
- 使用 `httpx.Client()` 调用 OpenAI 兼容 API
- 超时 15s，失败时返回空结果（不阻塞检索）
- 评估响应格式：
  ```json
  {"rag_confidence_score": 9.2, "evaluation": "highly_relevant"}
  ```

### 5.5 `models/reflection_rule.py`

```python
class PreventionRule(BaseModel):
    rule_number: int
    error_category: str
    preventive_rule: str
    reflection_text: str
    created_at: str
```

### 5.6 `models/rag_eval_result.py`

```python
class RAGEvalResult(BaseModel):
    rag_confidence_score: float | None = None
    evaluation: str | None = None
```

## 七、 设计约束与安全机制

### 7.1 stdout 绝对纯净

`stdout_guard.py` 以 `_MCPStdoutProxy` 拦截所有非 MCP 协议的 stdout 输出。新增的 `httpx` 调用使用 `Client(timeout=...)` 不会产生 stdout 污染。

### 7.2 异步事件循环保护

所有同步 I/O（Embedding 推理、LanceDB 写入、Markdown 文件写入）均通过 `loop.run_in_executor()` 分发至 `ThreadPoolExecutor`，确保 MCP 主事件循环不被阻塞。

### 7.3 火中取栗的异步钩子

保存工具的异步 Embedding + LanceDB 写入采用 `asyncio.ensure_future()` 触发，内部以 `try/except` 包裹，异常仅写入日志。Markdown 归档已在这之前完成，即使后台任务失败，数据也不会丢失。

### 7.4 RAG 评估的优雅降级

RAG 评估器在 LLM 调用超时、API Key 未配置、网络不可用时均返回空结果（两个字段均为 `None`），检索结果本身不受影响。
