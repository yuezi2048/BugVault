### 二、 核心依赖流向与层级规约 (Dependency Rules)

为了防止代码随着时间推移变成“意大利面条”，这套架构在层级调用上有极其严格的**单向依赖规约**：

1. **数据模型无处不在：** `models/bug_record.py` 是最底层的契约，所有层都可以引入它。
2. **严格的调用方向：** `main.py` -> `mcp_tools` -> `services` -> `database`。
   - `mcp_tools` 绝对不可以直接导入 `database`。它只能呼叫 `services`。
   - `database` 层绝对不知道 MCP 协议的存在，它只接受普通的 Python 字符串或 `BugRecord` 对象。
3. **依赖注入 (Dependency Injection)：** 在 `main.py` 启动时，初始化 `lancedb_client`，然后将其作为参数传给 `services`。这样在写测试用例时，你可以非常轻松地传入一个假的（Mock）数据库客户端，而不需要真的在本地建库。

### 三、 重点文件职责“防坑”演示

为了让你更直观地感受这套架构如何解决之前提到的工程化痛点，我们来看几个关键文件的代码伪骨架：

#### 1. `utils/logger.py` (防 stdout 污染的生命线)

MCP 协议依赖标准输出通信。这个文件是系统的“保安”。

```python
import sys
import logging

def setup_mcp_logger():
    """强制将所有日志输出到 stderr，保护 MCP 协议所需的 stdout"""
    logger = logging.getLogger("bugvault")
    logger.setLevel(logging.INFO)
    
    # 强制绑定到 sys.stderr
    handler = logging.StreamHandler(sys.stderr)
    formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
    handler.setFormatter(formatter)
    logger.addHandler(handler)
    
    return logger
```

#### 2. `mcp_tools/retrieve_tool.py` (极简门面模式)

在这个文件里，你看不到任何数据库操作，它只负责接客和参数转换。

```python
from services.retrieval_svc import RetrievalService
from pydantic import BaseModel

class RetrieveArgs(BaseModel):
    query: str
    tech_stack: str = None

async def handle_retrieve(args: dict, retrieval_svc: RetrievalService) -> str:
    """被 MCP Server 路由调用的入口函数"""
    # 1. 验证参数
    parsed_args = RetrieveArgs(**args)
    
    # 2. 调用核心服务层 (业务逻辑与截断在这里发生)
    compressed_results = await retrieval_svc.search_and_format(
        query=parsed_args.query, 
        tech_stack=parsed_args.tech_stack
    )
    
    # 3. 返回给 Claude
    return compressed_results
```

#### 3. `services/ingestion_svc.py` (智能追问的大脑)

这里是拦截大模型幻觉、防止垃圾数据入库的关键。

```python
from models.bug_record import BugRecord
from database.lancedb_client import LanceDBClient

class IngestionService:
    def __init__(self, db_client: LanceDBClient):
        self.db_client = db_client

    async def validate_and_save(self, raw_data: dict) -> str:
        # 1. 强校验：如果缺少关键字段，直接返回一段话指导 Claude 去追问用户
        if not raw_data.get("final_solution") or raw_data.get("final_solution") == "无":
            return "❌ 保存失败。请向用户询问缺失的【最终有效修复方案】，获取后再重新调用保存工具。"
        
        # 2. 校验通过，装配模型并落盘
        record = BugRecord(**raw_data)
        await self.db_client.insert_record(record)
        
        return "✅ Bug 经验已成功沉淀入本地知识库。"
```
