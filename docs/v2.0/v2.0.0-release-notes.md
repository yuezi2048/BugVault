# BugVault v2.0.0-beta Release Notes

> **"From code bugs to project conventions — AI should only need one reminder per rule."**

## Overview

v2.0.0 extends BugVault from "code bug memory" to **"project convention memory"**. The same retrieval pipeline (chunk-level dual recall, RRF fusion, Cross-Encoder reranking) now works for both bug experiences and project conventions — architecture rules, business rules, test conventions, and style guides.

**Mission: "Catch every mistake once, never repeat it."** — applied to every project rule, not just crashes.

## New in v2.0

### 🏷️ Convention Memory (`save_convention` / `retrieve_convention`)

Two new MCP tools that store and retrieve **project conventions** — rules the AI must follow when working on this project.

| Tool | Purpose | Example |
|------|---------|---------|
| `save_convention` | Persist a project rule after user correction | API response must be `{"code":0, "data":..., "message":"ok"}` |
| `retrieve_convention` | Proactively query conventions before making changes | Check if there's a test naming rule before creating a file |

**ConventionRecord** model — 4 mandatory fields:

| Field | Description |
|-------|-------------|
| `convention_name` | Short rule name (e.g. "API 响应统一格式") |
| `trigger_context` | When/where this applies (e.g. "在编写 REST API 端点时") |
| `incorrect_behavior` | What the AI should NOT do (violation example) |
| `correct_behavior` | What the AI SHOULD do (the rule) |

### 🔄 Shared Table Architecture

Conventions share the **same database tables** as bug records, distinguished by `record_type`:

| Table | `record_type='bug'` | `record_type='convention'` |
|-------|---------------------|---------------------------|
| `bug_records` (parent) | Existing bug experiences | Convention metadata |
| `bugvault_chunks` (chunks) | Error/semantic chunks | Context/correct-behavior chunks |

This means conventions benefit from the **same retrieval pipeline** as bugs: vector ANN → FTS BM25 → RRF fusion → Cross-Encoder reranking, all filtered by `record_type='convention'`.

### 📚 Markdown Archive

Conventions are archived to `~/.bugvault/archive/conventions/` as human-readable Markdown, separate from bug archives.

### 🗄️ Schema Auto-Migration

Existing v1.1.x databases lack the `record_type` column. v2.0 adds an automatic migration:

- **Automatic**: `_migrate_v2_schema_if_needed()` runs at server startup in `_init_table()` and `_init_chunks_table()`
- **Idempotent**: Only adds the column if missing; safe to run multiple times
- **Non-destructive**: All existing bug records get `record_type='bug'` — no data loss
- **Manual fallback**: `uv run python scripts/migrate_v2.py` for standalone execution

## Upgrade Path from v1.1.x

```bash
# 1. Pull latest code
git checkout v2.0.0

# 2. (Optional) Manual migration — server auto-migrates on startup,
#    but you can run this standalone
uv run python scripts/migrate_v2.py

# Expected output:
#   bug_records: 2114 records, all record_type='bug'
#   bugvault_chunks: 5377 chunks, all record_type='bug'

# 3. Restart the MCP server — done
```

## Verification

Run the end-to-end test suite:

```bash
uv run pytest tests/test_core.py -v
```

**Tested scenarios:**
- Save convention → verify `record_type='convention'` in DB
- Vector ANN search with `filter_clause="record_type='convention'"`
- FTS BM25 search with `filter_clause="record_type='convention'"`
- Parent-record batch fetch from chunk hits
- Cross-Encoder reranking on convention results
- Idempotent schema migration (re-run is no-op)
- Legacy bug records remain intact with `record_type='bug'`

## Known Limitations

| Issue | Impact | Workaround |
|-------|--------|------------|
| Cross-Encoder 1.1GB model cold start | First retrieval takes ~15-30s | Model is singleton; subsequent calls ~250ms |
| Local LanceDB per user | Team conventions not shared | Mount `~/.bugvault/` on shared storage or use CI integration |
| No human-readable convention dashboard | Only AI can query conventions | Archives available at `~/.bugvault/archive/conventions/` |

## Configuration

No new environment variables — conventions use the same `BUGVAULT_*` settings as bug records (embedding model, retrieval params, etc.).
